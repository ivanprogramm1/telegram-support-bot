# Support-бот (регистрация + верификация + постбеки Pocket Partners)

## Что делает бот

- Приветствие → уточнение опыта → ссылка на регистрацию (по ГЕО)
- Приём UID, проверка в базе (база наполняется автоматически через постбеки)
- Если UID найден — присылает 3 скриншота-инструкции по верификации
- После верификации — сообщает, что дальнейшие шаги обсуждаются лично, и даёт ссылку на вас
- `/faq` — направляет в раздел «Помощь» на самой платформе + ссылка на вас, если не нашли ответ
- `/risks` — честный текст про риски бинарных опционов
- `/support` (или кнопка) — эскалация на вас в любой момент
- Любое неожиданное сообщение — тоже уходит на эскалацию, а не теряется

## Что НЕ входит в бота (сознательно)

- Рассылка скриншотов чужих балансов/чатов
- Текст "как я начинал, увеличивал депозит"
- Автоматическая отправка ссылок на закрытые каналы/голосовых после депозита
- Разблокировка "уровней" по сумме депозита

Всё это, если решите использовать — ведёте сами, лично, после того как бот передал вам человека через эскалацию. Точки для встраивания этого в код я специально не оставлял (ни как заглушки, ни как TODO) — это было осознанное решение, а не недоработка.

## Структура проекта

```
pocket_support_bot/
├── main.py          # точка входа: запускает бота + постбек-сервер
├── bot.py           # логика Telegram-бота (aiogram)
├── postback.py       # приём постбеков от Pocket Partners (FastAPI)
├── db.py             # работа с базой трейдеров (SQLite)
├── texts.py           # все тексты сообщений — правьте смело
├── config.py          # переменные окружения
├── requirements.txt
├── .env.example
└── assets/
    ├── verification_1.jpg
    ├── verification_2.jpg

```

## Локальный запуск (для теста перед деплоем)

```bash
cd pocket_support_bot
pip install -r requirements.txt
cp .env.example .env
# впишите в .env реальный BOT_TOKEN и остальные значения
python3 main.py
```

Бот запустится в режиме long polling, постбек-сервер поднимется на порту 8080 (проверить: `curl http://localhost:8080/` → `{"status":"ok"}`).

## Асеты (скриншоты и голосовые)

Папка `assets/` содержит:
- `verification_1.jpg`, `verification_2.jpg`, `verification_3.jpg` — скрины по прохождению верификации
- **`workflow_1.jpg`, `workflow_2.jpg`, `workflow_3.jpg`, `workflow_4.jpg`** — скрины работы в каналах (как проходят сессии, как смотрится интерфейс каналов и т.п.). Отправляются пользователю после подтверждения депозита, перед голосовыми про риски. Это визуальная часть — пользователь может сразу понять, подходит ли ему стиль работы.
- **`voice_risks.ogg`** — голосовое о рисках бинарных опционов и финансовом риск-менеджменте
- **`voice_rules.ogg`** — голосовое о правилах платформы и вашего сообщества (например: один аккаунт, что будет при нарушениях и т.п.)

Как добавить новые скрины/голосовые: просто положите их в папку `assets/` с соответствующими именами и задеплойте на Railway. Бот автоматически подхватит их при запуске.

## Поток регистрации пользователя (финальный честный вариант)

1. **Приветствие → Опыт → ГЕО → UID** (сохраняется, вы узнаёте об этом)
2. **Возраст** (18+)
3. **Верификация** (3 скрина по прохождению KYC на платформе)
4. **Депозит** (ждём, пока пользователь пополнит баланс, потом нажмёт кнопку)
5. **Стратегия торговли** (честное объяснение: 1–5% за сделку, без историй про личный опыт)
6. **Workflow в каналах** (4 скрина того, как выглядит работа, чтобы человек понял, подходит ли ему)
7. **Предупреждения** (голосовое про риски и про правила платформы)
8. **Выбор канала** (Семья Макса или ОТС; ОТС только если баланс ≥ 200$)
9. **Эскалация на вас** (дальше всё лично)

Все тексты можно редактировать в `texts.py`, все асеты лежат в папке `assets/`. Бот не совершает никаких "дожимов" — просто предоставляет информацию и передаёт человека вам, когда регистрация завершена.

## Деплой на Railway (пошагово)

1. Зарегистрируйтесь на railway.app, подключите GitHub (или используйте Railway CLI для загрузки прямо из этой папки)
2. Создайте новый проект → Deploy from GitHub repo (залейте эту папку в свой репозиторий) или `railway up` через CLI
3. В разделе **Variables** добавьте:
   - `BOT_TOKEN` — токен вашего бота
   - `SUPPORT_USERNAME` — ваш юзернейм без @ (сейчас `max_supportTraide`)
   - `POSTBACK_SECRET` — придумайте случайную строку (например, из генератора паролей)
   - остальные переменные из `.env.example`, если нужно поменять ссылки
4. Railway автоматически определит `PORT` — ничего дополнительно указывать не нужно
5. Убедитесь, что в настройках проекта указана команда запуска: `python3 main.py` (обычно Railway подхватывает это сам по `requirements.txt`, но проверьте в Settings → Deploy)
6. После деплоя Railway выдаст публичный домен вида `https://ваш-проект.up.railway.app`
7. **Важно:** база данных SQLite (`traders.db`) хранится на диске контейнера. На бесплатном плане Railway диск не персистентный между редеплоями — то есть при каждом новом деплое база обнулится. Для реальной эксплуатации подключите Railway Volume (Settings → Volumes → Add Volume, примонтируйте, например, в `/data`, и поменяйте `DB_PATH=/data/traders.db` в переменных) — иначе первый же редеплой сотрёт всех накопленных трейдеров.

## Настройка постбеков в кабинете Pocket Partners

Зайдите в раздел «Постбэки» → «Глобальный постбек» → включите нужные события и вставьте URL:

**Регистрация:**
```
https://ВАШ-ДОМЕН.up.railway.app/postback?event=reg&trader_id={trader_id}&country={country}&date_time={date_time}&link_type={link_type}&secret=ВАШ_POSTBACK_SECRET
```

**Подтверждение email:**
```
https://ВАШ-ДОМЕН.up.railway.app/postback?event=email_confirm&trader_id={trader_id}&secret=ВАШ_POSTBACK_SECRET
```

**FTD (первый депозит):**
```
https://ВАШ-ДОМЕН.up.railway.app/postback?event=ftd&trader_id={trader_id}&sumdep={sumdep}&date_time={date_time}&secret=ВАШ_POSTBACK_SECRET
```

**Повторный депозит:**
```
https://ВАШ-ДОМЕН.up.railway.app/postback?event=redeposit&trader_id={trader_id}&sumdep={sumdep}&secret=ВАШ_POSTBACK_SECRET
```

**Комиссия:**
```
https://ВАШ-ДОМЕН.up.railway.app/postback?event=commission&trader_id={trader_id}&commission={commission}&secret=ВАШ_POSTBACK_SECRET
```

**Вывод средств:**
```
https://ВАШ-ДОМЕН.up.railway.app/postback?event=withdrawal&trader_id={trader_id}&wdr_sum={wdr_sum}&status={status}&secret=ВАШ_POSTBACK_SECRET
```

## Логика верификации (обновлено)

После того как UID найден в базе:

1. Бот спрашивает про возраст (18+). Если "нет" — сразу эскалация на вас, дальше бот ничего не делает
2. Если "да" — бот объясняет важность верификации (нужно загрузить документ, без этого не получится торговать/выводить), присылает 3 скриншота-инструкции
3. Появляется кнопка «Прошёл верификацию ✅» — при нажатии бот передаёт человека вам напрямую, без дополнительной автоматической проверки (Pocket Partners не присылает отдельный постбек на это событие, поэтому финальную проверку статуса вы делаете лично в переписке)

Вам не нужно ничего вручную отмечать в самом боте — весь процесс работает без дополнительных действий с вашей стороны.



Проще всего — открыть базу напрямую (через Railway shell: `sqlite3 traders.db "SELECT * FROM traders WHERE uid='UID_ЗДЕСЬ';"`). Если нужен отдельный `/check UID`-хендлер в самом боте, доступный только вам по вашему Telegram ID — скажите, это несложно добавить отдельным шагом.
